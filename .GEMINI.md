# .GEMINI.md - browserless-tooling

This project provides idempotent Bash scripts for provisioning a hardened Browserless Chromium instance and a corresponding Node.js MCP wrapper. It does not contain compiled code.

### Key Scripts
- `aibrowse-setup.sh`: Deploys the core Browserless Chromium service.
- `browsewrap-setup.sh`: Deploys a Node.js wrapper that connects to an existing Browserless instance.

### Workflow
1.  Run `aibrowse-setup.sh` to create a new Browserless instance.
2.  Run `browsewrap-setup.sh` with the same instance name to deploy the wrapper.

### Core Commands
Both scripts require `sudo` because they manage system services (Docker) and firewall rules.
- `sudo bash aibrowse-setup.sh <instance-name>`:
    - Provisions a new Browserless Chromium service under `/docker/aibrowse/<instance-name>`.
    - Generates a `docker-compose.yml`, `.env` file with a secure token, and client examples.
    - Opens a random, high-numbered port in the firewall (`firewalld`).
    - Starts the service using `docker compose`.
- `sudo bash browsewrap-setup.sh <instance-name>`:
    - Provisions a Node.js wrapper service under `/docker/browsewrap/<instance-name>`.
    - Reads configuration from the corresponding `aibrowse` instance.
    - Generates its own Dockerfile and `docker-compose.yml`.
    - Starts the service and updates a central MCP config file at `~/mcp.json`.

### Validation
- `shellcheck *.sh`: Check for shell scripting errors.
- `bash -n *.sh`: Validate the syntax of the scripts without executing them.
