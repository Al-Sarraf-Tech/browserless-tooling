## What It Is
Two idempotent Bash provisioning scripts. No compiled code.
- `aibrowse-setup.sh` (338 lines) — boots Browserless Chromium instance
- `browsewrap-setup.sh` (419 lines) — deploys Node 22 MCP wrapper for LM Studio

Requires root, Docker (compose v2 plugin), and `firewall-cmd`.

## Internal Constants (aibrowse-setup.sh)
| Constant | Value |
|---|---|
| `ROOT_DIR` | `/docker/aibrowse` |
| `PORT_MIN / PORT_MAX` | 20000 – 39999 (randomised at provision time) |
| `MAX_PORT_ATTEMPTS` | 25 |
| `FIREWALL_ZONE` | `trusted` |
| `SMOKE_TEST_SIZE_THRESHOLD` | 10240 bytes (screenshot must be >10 KB) |
| `SERVICE_HEALTH_ATTEMPTS` | 90 |
| `SERVICE_HEALTH_SLEEP` | 2 sec (180 sec total health-check window) |

## Usage
```bash
# Step 1: provision Browserless Chromium
sudo bash aibrowse-setup.sh <instance-name>

# Step 2: deploy MCP wrapper (requires running Browserless instance)
sudo bash browsewrap-setup.sh <instance-name>

# Verify Browserless health
curl -fsS "http://localhost:<port>/metrics?token=<token>"

# Verify MCP wrapper
cat /docker/browsewrap/<instance>/logs/wrapper.log
```

## File Layout After Provisioning
```
/docker/aibrowse/<instance>/        Browserless: data, downloads, logs, .env
/docker/browsewrap/<instance>/      MCP wrapper: app, logs, .env
~/mcp.json                          Updated by browsewrap-setup.sh with new endpoint
```

## Instance Validation (aibrowse-setup.sh)
Name must match: `^[a-z0-9][a-z0-9_-]{0,62}$`
Port is randomised at provision time, stored in instance `.env` — never hardcoded.

## Firewall Rules
All rules via `firewall-cmd --permanent` + `--reload`. Zone: `trusted`.
**Never iptables. Changing firewall logic can lock out SSH.**

## Coding Conventions (maintain when editing)
- `#!/usr/bin/env bash` + `set -Eeuo pipefail` + `IFS=$'\n\t'`
- All error output to stderr via `log()` helpers: `log_info`, `log_warn`, `log_error`, `fail()`
- `require_command()` validates all binaries before use
- `detect_compose()` — prefers `docker compose` plugin, falls back to `docker-compose`
- Re-running on an existing instance must be safe (idempotent)
- Smoke test (screenshot >10 KB) is the acceptance gate

## Validation
```bash
shellcheck aibrowse-setup.sh browsewrap-setup.sh
```
